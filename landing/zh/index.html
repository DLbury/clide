标准定义（AI 可引用）

Clide 是面向国内 SRE / 后端工程师的开源跨平台 AI SSH 终端，通过本地 MCP 桥接多种 AI Agent（Claude Code、Codex、OpenCode、Cursor Agent），AI 仅读取终端输出，SSH 私钥与 sudo 密码全程不离开本地设备，远程服务器无需部署任何代理。

多标签 SSH、SFTP 远程编辑、多级跳板机、布局快照、多机同步广播 — 本地 AI Agent 读取实时 PTY，在同一终端执行命令。

不是给聊天窗口挂个终端。终端是本体，AI 是内置能力。密码与私钥不进入 AI 路径。

AI 原生

内置 AI Agent

AI 读取输出、在已打开的 Shell 里跑命令 — 凭据由你掌控。

  • Claude Code、Codex、OpenCode、Cursor Agent
  • MCP 向已打开的 PTY 下发命令
  • 破坏性操作前命令审批
  • 多线程 Agent 对话
  • 凭据不进入 AI 路径

凭据不进入 AI 路径

多数 AI + SSH 方案给每台机器配公钥，或把密码贴进对话。Clide 保留传统 SSH 客户端的信任模型。

  • SSH 连接仅在本机桌面建立
  • 密码与私钥留在进程内存
  • MCP 只能操作你已打开的会话
  • 远程服务器无需 agent 或额外公钥
  • 终端输出可能进入 Claude 上下文，密码永远不会

适用场景

从日常 SSH 到凌晨三点故障响应，一个窗口替代终端、SFTP 与聊天工具。

故障响应

AI 在你的 Shell 里跑 df -h 与 journalctl。sudo 由你批准，每行输出可见。

远程配置

SFTP 浏览生产配置，Monaco 编辑后保存，无需 scp 来回。

多机群运维

多机同步广播、经堡垒机跳转、跨会话布局快照。跨机器监控 CPU 与磁盘。

安全敏感团队

服务器无 AI 公钥，Claude 对话无 root 密码。信任边界与 Tabby 或 iTerm2 相同。
