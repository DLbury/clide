AI 能操作服务器 —— 但不持有你的凭据

多数 AI + SSH 方案的困境：要么给每台机器配 AI 公钥，要么把密码贴进对话。Clide 的本地中转架构保留传统 SSH 客户端的安全模型。

  • SSH 连接仅在本机桌面建立与维护
  • 密码与私钥不离开进程内存
  • MCP 只能操作你已打开并登录的会话
  • 远程服务器无需 agent 或额外 SSH 公钥
  • 终端输出文本可能进入 Claude 上下文 —— 密码永远不会

常见问题

关于 Clide 作为 SSH 终端、SFTP 客户端与 AI 运维工具的常见疑问。

Clide 是 AI 终端还是普通 SSH 客户端？

两者都是。Clide 首先是功能完整的跨平台 SSH 终端与 SFTP 客户端 —— 多标签 Shell、远程文件编辑、服务器监控、跳板机。AI 层（Claude Code via MCP）可选：向已打开的 Shell 下发命令并读回输出，不替代或绕过 SSH 客户端。

终端输出会上传到 Anthropic 吗？

会 —— 你让 Claude 读取的输出文本会成为 Claude Code 对话的一部分，与任何 Claude Code 用法相同。永远不会上传的：SSH 密码、私钥、sudo 密码。不要把 secret 贴进 AI 面板，在 Shell 里输入敏感信息。

为什么不直接用 Tabby 或 iTerm2，另开 Claude 窗口？

可以，但 Claude 看不到你的实时终端，只能手工复制粘贴。Clide 的价值在于 AI 读取真实 PTY 输出并把命令写回同一条 Shell，加上 SFTP、Monaco 编辑与监控，全在一个窗口。

支持哪些平台？

Windows 10+、macOS 11+ 及主流 Linux 发行版。基于 Tauri 2 原生性能。GitHub Releases 提供预编译安装包。

Clide 和 AITerm 是什么关系？

Clide 的 MCP 服务器名为 aiterm —— 负责把 Claude Code 桥接到你的终端会话。桌面应用品牌是 Clide；仓库中 MCP 工具以 AITerm 命名。
