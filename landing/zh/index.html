凭据不进入 AI 路径

多数 AI + SSH 方案给每台机器配公钥，或把密码贴进对话。Clide 保留传统 SSH 客户端的信任模型。

  • SSH 连接仅在本机桌面建立
  • 密码与私钥留在进程内存
  • MCP 只能操作你已打开的会话
  • 远程服务器无需 agent 或额外公钥
  • 终端输出可能进入 Claude 上下文，密码永远不会

适用场景

从日常 SSH 到凌晨三点故障响应，一个窗口替代终端、SFTP 与聊天工具。

故障响应

AI 在你的 Shell 里跑 df -h 与 journalctl。sudo 由你批准，每行输出可见。

远程配置

SFTP 浏览生产配置，Monaco 编辑后保存，无需 scp 来回。

多机群运维

标签页会话、经堡垒机跳转，跨机器监控 CPU 与磁盘。

安全敏感团队

服务器无 AI 公钥，Claude 对话无 root 密码。信任边界与 Tabby 或 iTerm2 相同。
