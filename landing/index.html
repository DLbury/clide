AI That Operates Servers — Without Owning Your Credentials

The problem with most AI + SSH setups: you either distribute AI public keys to every server, or paste passwords into chat. Clide's local relay architecture keeps the security model of a traditional SSH client.

  • SSH connections established and maintained only on your desktop
  • Passwords and private keys never leave process memory
  • MCP operates only on sessions you've already opened and logged into
  • No agent software or extra SSH keys on remote servers
  • Terminal output text may enter Claude context — passwords never do

Frequently Asked Questions

Common questions about Clide as an SSH terminal, SFTP client, and AI ops tool.

What is Clide — an AI terminal or a regular SSH client?

Both. Clide is a full-featured cross-platform SSH terminal and SFTP client first — multi-tab shells, remote file editing, server monitoring, jump hosts. The AI layer (Claude Code via MCP) is optional: it sends commands to your already-open shell and reads output back, without replacing or bypassing your SSH client.

Does terminal output get sent to Anthropic?

Yes — the text output you let Claude read becomes part of your Claude Code conversation, same as any Claude Code session. What's never sent: SSH passwords, private keys, and sudo passwords. Don't paste secrets into the AI panel; use the shell for sensitive input.

Why not just use Tabby or iTerm2 with Claude in another window?

You can, but Claude can't see your live terminal — you copy-paste by hand. Clide's value is the AI reads the real PTY output and runs commands back into the same shell, plus SFTP, Monaco editing, and monitoring in one window.

Which platforms does Clide support?

Windows 10+, macOS 11+, and major Linux distributions. Built with Tauri 2 for native performance. Prebuilt installers available on GitHub Releases.

Is Clide related to AITerm?

Clide's MCP server is named aiterm — it bridges Claude Code to your terminal sessions. The desktop app brand is Clide; the MCP tooling lives under the AITerm name in the repository.
