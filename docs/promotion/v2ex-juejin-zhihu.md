## 一、V2EX（节点：分享创造 / 程序员）

> V2EX 忌纯广告，标题要朴素，正文先讲问题再讲方案，最后附repo。不要在标题里写「开源」「求star」。

**标题：**
```
做了一个 SSH 终端，让 Claude Code 帮你排障但碰不到你的密码
```

**正文：**

> 先说自己遇到的问题：想让 Claude Code 帮我半夜查服务器，但现有的玩法都不太敢用——
>
> - 给每台机器配 AI 公钥：AI 端一旦泄露，所有机器一起沦陷
> - 把 root 密码贴给 Claude：凭据离开本机，还得上传到 Anthropic
> - 配无密码 sudo：安全审计直接红牌
>
> 于是做了 Clide：一个桌面 SSH 客户端，凭据全部留在本机，本地的 Claude Code 通过 MCP 把命令下发到你**已经登录**的那条左侧 Shell（同一条 PTY，和你手敲一样）。需要 sudo 时，提示符出现在左侧 Shell，**你自己输密码，AI 看不到**。服务器上不用装任何 agent，不用配额外公钥。
>
> 同一个窗口里还有：多标签分屏终端、SFTP 文件浏览、Monaco 编辑远程配置、CPU/内存/磁盘监控（走独立 exec 通道，不挡你的交互 Shell）。
>
> 技术栈：Tauri 2 + Rust（russh、portable-pty），前端 Next.js。Windows / macOS / Linux 都有安装包。
>
> 仓库（MIT，可直接下安装包）：https://github.com/DLbury/clide
>
> 想听运维同学的真话：这个信任模型够不够？离让你放心给实习生用还差什么？与其给个假安全感，我宁可现在被喷。

**跟帖回复模板（前几条要快）：**

> 补一下威胁模型，免得被问爆：你让 Claude 看到的**终端输出**会进入 Claude Code 对话（这跟任何 Claude Code 用法一样）。但**永远不会**进 AI 路径的是：SSH 密码、私钥、sudo 密码——这些都敲在 xterm 里，不敲在对话框。所以别把 secret 粘进右侧 AI 面板就行。
